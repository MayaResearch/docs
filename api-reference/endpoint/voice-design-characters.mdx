<Note>
  This endpoint returns a list of all available pre-designed voice characters. Each character has been optimized for specific use cases and provides consistent, high-quality voice output.
</Note>

<ParamField header="X-API-Key" type="string" required placeholder="maya_YOUR_API_KEY_HERE">
  API key for authentication
</ParamField>

## Response

<ResponseField name="characters" type="array">
  Array of character objects containing voice information
  
  <Expandable title="Character Object">
    <ResponseField name="character_id" type="string">
      Unique identifier for the character (e.g., "m4a7b2")
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Display name of the character
    </ResponseField>
    
    <ResponseField name="description" type="string">
      Detailed description of the voice characteristics
    </ResponseField>
    
    <ResponseField name="gender" type="string">
      Voice gender: "male", "female", or "neutral"
    </ResponseField>
    
    <ResponseField name="age_range" type="string">
      Approximate age range (e.g., "20s", "30s", "40s")
    </ResponseField>
    
    <ResponseField name="accent" type="string">
      Voice accent (e.g., "american", "british", "indian")
    </ResponseField>
    
    <ResponseField name="use_cases" type="array">
      Recommended use cases for this character (e.g., ["podcast", "customer_service"])
    </ResponseField>
    
    <ResponseField name="sample_url" type="string">
      URL to a sample audio file demonstrating this character's voice
    </ResponseField>
  </Expandable>
</ResponseField>

## Using Character Data

Once you retrieve the character list, you can use the `character_id` to generate speech:

<CodeGroup>

```bash cURL
curl --location 'https://v3.mayaresearch.ai/v1/tts/generate' \
--header 'Content-Type: application/json' \
--header 'X-API-Key: maya_YOUR_API_KEY_HERE' \
--data '{
  "character_id": "mfbo7r",
  "text": "Welcome back to another episode of our podcast! Today we are diving into an absolutely fascinating topic.",
  "stream": false
}'
```

```javascript JavaScript
const response = await fetch('https://v3.mayaresearch.ai/v1/tts/generate', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-API-Key': 'maya_YOUR_API_KEY_HERE'
  },
  body: JSON.stringify({
    character_id: "mfbo7r",
    text: "Welcome back to another episode of our podcast! Today we are diving into an absolutely fascinating topic.",
    stream: false
  })
});

const audioData = await response.blob();
```

```python Python
import requests

url = "https://v3.mayaresearch.ai/v1/tts/generate"
headers = {
    "Content-Type": "application/json",
    "X-API-Key": "maya_YOUR_API_KEY_HERE"
}
payload = {
    "character_id": "mfbo7r",
    "text": "Welcome back to another episode of our podcast! Today we are diving into an absolutely fascinating topic.",
    "stream": False
}

response = requests.post(url, json=payload, headers=headers)
```

## Character Selection Tips

<AccordionGroup>
  <Accordion title="For Customer Service" icon="headset">
    Look for characters with:
    - Friendly, approachable descriptions
    - Clear, professional tone
    - Neutral or warm characteristics
    - "customer_service" or "assistant" in use_cases
  </Accordion>
  
  <Accordion title="For Podcasts & Content" icon="podcast">
    Look for characters with:
    - Conversational, engaging style
    - Appropriate age and accent for your audience
    - "podcast", "presenter", or "host" in use_cases
    - Natural, expressive delivery
  </Accordion>
  
  <Accordion title="For Audiobooks & Narration" icon="book-open-reader">
    Look for characters with:
    - Clear, consistent delivery
    - Appropriate tone for your content
    - "audiobook" or "narration" in use_cases
    - Good range for character voices
  </Accordion>
  
  <Accordion title="For Professional Content" icon="briefcase">
    Look for characters with:
    - Authoritative, confident tone
    - Professional, polished delivery
    - "professional", "presentation", or "formal" in use_cases
    - Clear articulation
  </Accordion>
</AccordionGroup>

## Filtering and Sorting

You can filter and sort characters in your application based on the response data:

```javascript JavaScript Example
// Filter by gender
const maleCharacters = characters.filter(c => c.gender === 'male');

// Filter by use case
const podcastCharacters = characters.filter(c => 
  c.use_cases.includes('podcast')
);

// Filter by accent
const americanCharacters = characters.filter(c => 
  c.accent === 'american'
);

// Sort by age range
const sortedByAge = characters.sort((a, b) => 
  a.age_range.localeCompare(b.age_range)
);
```

## Rate Limits

<Info>
  **Rate Limits for Character Listing:**
  - 100 requests per minute
  - This endpoint is optimized for caching
  
  Consider caching the character list in your application as it updates infrequently.
</Info>

## Error Codes

| Code | Description | Resolution |
|------|-------------|------------|
| 401 | Authentication failed | Verify your API key is correct |
| 403 | Access denied | Check API permissions |
| 429 | Rate limit exceeded | Implement caching and retry logic |
| 500 | Internal server error | Contact support if persistent |

<Warning>
  Cache the character list in your application to reduce API calls and improve performance.
</Warning>
